## 📝 文档维护规范

### 文档更新原则
- **及时更新**: 代码变更后及时更新相关文档
- **保持同步**: 确保文档与实际实现保持一致
- **版本控制**: 重要变更记录在相应的更新日志中

### 贡献指南
1. 文档内容应清晰、准确、简洁
2. 使用 Markdown 格式编写
3. 重要变更前先讨论，确保文档结构合理
4. 定期review和更新过期内容

### 文档版本说明
- **v1.0** - 初始版本，包含基础业务和技术设计
- **v1.1** - 权限体系重构，3层权限架构
- **v1.2** - 文档架构重组，分类管理
