## 🎯 项目概述

### 功能定位
共读活动模块是QQClub的核心功能之一，旨在组织和管理多人参与的阅读活动，通过结构化的流程确保阅读活动的顺利进行和有效管理。

### 核心价值
- **结构化阅读**: 提供有序的阅读计划和时间管理
- **社交互动**: 支持参与者之间的交流和讨论
- **激励机制**: 通过打卡和小红花系统激发参与积极性
- **进度管理**: 实时跟踪参与者的阅读进度和完成情况

### 功能范围
- ✅ 分步骤创建共读活动（3步流程）
- ✅ 多种活动模式支持（笔记打卡、自由讨论、视频会议、线下交流）
- ✅ 灵活的费用设置（免费、押金制、收费制）
- ✅ 智能领读人分配机制
- ✅ 完善的打卡规则和统计系统
- ✅ 参与证书生成系统
- ✅ 开放的评论和交流环境

### 模块独立性设计
**架构原则**: 共读区与交流区完全解耦，仅共享用户系统

**独立性体现**:
- **数据隔离**: 共读活动和论坛帖子无业务关联
- **权限独立**: 各自的权限管理体系
- **功能独立**: 完全独立的功能模块，无跨模块依赖
- **用户共享**: 仅在用户层面统一，其他业务完全分离

## 👥 用户角色

| 角色 | 权限描述 | 特殊功能 |
|------|----------|----------|
| **普通用户** | 查看活动、可报名或围观 | 无 |
| **参与者** | 已报名用户，可打卡、评论、报名领读 | 参与讨论、接收小红花 |
| **围观者** | 未报名用户，仅可评论作业 | 基础互动权限 |
| **领读人** | 指定的每日负责人，可发小红花 | 领读内容管理、点评权限 |
| **小组长** | 活动创建者，拥有最高权限 | 审批报名、活动管理、结算权限 |
| **管理员** | 平台管理员 | 全平台活动管理权限 |

## 🔄 费用模式

### 免费模式
- 无任何费用要求
- 零门槛参与
- 适合试水和体验

### 押金制模式
- 20%给小组长作为固定报酬，不参与退还分配
- 80%费用进入押金池，根据完成率进行退还
- 简单粗暴的退还机制，无部分退还

### 收费制模式
- 收费不退，全部费用归小组长所有
- 适合高质量、有附加值的活动
- 小组长需要提供更多价值服务

## 🏗️ 技术架构

- **前端**: 微信小程序 (WeChat Mini-Program)
- **后端**: Ruby on Rails 8
- **数据库**: SQLite (开发) / PostgreSQL (生产)
- **认证**: JWT Token
- **部署**: 支持云服务部署
